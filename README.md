# Requirement Dojo - 需求道场 🥋

一个沉浸式 Web 应用，通过模拟真实业务场景，训练产品经理的需求洞察力和需求分析能力。

## ✨ 核心特色

### 🎭 真实业务场景
- **5 个行业场景**：金融、医疗、电商、合规、运动健康
- **业务人员视角**：理财顾问、导诊护士、客服主管、审批专员、健身教练
- **渐进式信息释放**：从表面抱怨到深层原因，模拟真实的职场对话

### 💬 AI 驱动对话
- **DeepSeek v3.2 模型**：智能、自然的业务人员对话
- **两种难度模式**：
  - 初级难度：逻辑清晰，信息完整
  - 高级难度：表达破碎，需要挖掘，包含噪音信息

### 📊 五维能力评估
- 需求洞察力：是否挖掘出隐藏约束？
- 逻辑完备性：是否覆盖所有需求？
- AI 思维：是否提出有价值的 AI 方案？
- 文档专业度：表达是否精准、结构化？
- 方案可行性：是否考虑了技术和组织约束？

## 🎮 使用指南

### 训练流程

**Phase 1: 场景初始化**
- 选择行业（金融/医疗/电商/合规/运动）
- 选择难度（初级难度/高级难度）

**Phase 2: 需求访谈**
- 与 AI 业务人员对话
- 挖掘痛点和约束条件
- 提示：问"具体情况"、"为什么"、"有什么限制"

**Phase 3: 需求文档编写**
- 整理业务目标
- 列出痛点清单
- 定义核心功能

**Phase 4: 方案设计**
- 提交功能设计方案
- 思考 AI 结合点（必填）

**Phase 5: 五维复盘**
- 查看能力雷达图
- 阅读专家点评
- 参考高阶 AI 解决方案

## 💡 训练技巧

### 提问建议

❌ **不推荐的提问**
- "有什么问题？"（太泛）
- "能说说吗？"（不具体）

✅ **推荐的提问**
- "能具体说说有多严重吗？大概有多少客户这样？"
- "为什么会出现这种情况？客户是怎么反馈的？"
- "有什么公司规定或限制吗？"
- "现在用的系统怎么样？"
- "人手和时间够吗？"

### 需求挖掘路径

1. **建立信任** → "听说最近遇到了一些问题？"
2. **挖掘数据** → "能具体说说有多严重吗？"
3. **追问原因** → "为什么会这样？客户怎么说的？"
4. **识别约束** → "有规定吗？" "系统情况？" "资源够吗？"
